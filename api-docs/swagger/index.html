DASAM ARS 배치 주문 API

한 번의 API 호출로 여러 건의 결제 주문을 동시 처리하는 배치 시스템

verified Version 4.0.0 layers BATCH SUPPORT code JSON ONLY
api

API 개요

upload 요청 (Request)

  • 공통 파라미터 - 고객 정보, 카드 정보 (모든 주문에 적용)
  • orders 배열 - 개별 주문 목록 (주문번호, 금액, 상품명)

download 응답 (Response)

  • batch_summary - 배치 처리 요약 (총/성공/실패 건수)
  • req_result 배열 - 각 주문별 처리 결과 코드
POST /db/kicc_ars_order_v3_batch.asp
운영 서버 https://www.arspg.co.kr/ars/kicc/db/kicc_ars_order_v3_batch.asp
개발 서버 https://www.arspg.co.kr/ars/kicc/dev/db/kicc_ars_order_v3_batch.asp
description
Content-Type application/json
layers
권장 배치 크기 1~50건
input

입력 파라미터

check_circle 필수 파라미터

파라미터 타입 설명 예시
mode* string 요청 모드 (고정값: ars_data_add) ars_data_add
terminal_id* string 가맹점 터미널 ID (KICC 발급) T0021720
request_type* string ARS=전화결제 / SMS=문자인증 / KTK=카카오톡 ARS
cc_name* string 고객명 홍길동
phone_no* string 전화번호 (숫자만, 하이픈 제거) 01024020684
card_no* string 카드번호 (12자리) 123456789012
expire_date* string 유효기간 (YYMM 형식) 2512
install_month* string 할부개월 (00=일시불, 02~12=할부) 00
orders* array 주문 배열 (아래 참조) [...]

add_circle_outline 선택 파라미터

파라미터 타입 설명 예시
cc_email string 고객 이메일 user@example.com
sms_message string 사용자 정의 메시지 (SMS/KTK 전용, 최대 90자) 결제 안내 메시지
agency_name string 여행사명 (SMS/KTK에서 sms_message 없을 시 필수) 테스트10 여행사
reservation_no string 예약번호 (SMS/KTK에서 sms_message 없을 시 필수) ABC1234
alert_show string 응답 표시 여부 (Y/N) Y
warning SMS/KTK 조건부 필수 파라미터: sms_message가 제공되지 않으면 agency_name과 reservation_no가 필수입니다. sms_message가 있으면 무시됩니다.

list_alt orders 배열 항목

info orders 배열의 모든 항목에는 아래 3개 필드가 필수입니다.
파라미터 타입 필수 설명 예시
order_no string O 주문번호 (가맹점별 고유) ORD20250118001
amount number/string O 결제금액 (1원 이상) 50000
cc_pord_desc string O 상품명 프리미엄 서비스
output

응답 파라미터

summarize batch_summary (배치 처리 요약)

필드 타입 설명
total integer 총 주문 건수
success integer 성공 건수
fail integer 실패 건수

list_alt req_result (주문별 처리 결과 배열)

필드 타입 설명
order_no string 주문번호
phone_no string 전화번호
result_code string 결과 코드 (4자리, 0000=성공)
message string 결과 메시지
error_outline

에러 코드

check_circle 성공

코드 메시지 설명
0000 등록성공 주문이 성공적으로 생성됨

warning 파라미터 누락 에러

코드 메시지 누락 파라미터 해결 방법
0001 전송데이터구분오류 mode ars_data_add로 설정
0002 주문번호누락 order_no 유효한 주문번호 전달
0003 가맹점터미널ID누락 terminal_id KICC 발급 터미널ID 전달
0004 ARS타입누락 request_type ARS/SMS/KTK 값 전달
0005 카드번호누락 card_no 12자리 카드번호 전달
0006 고객명누락 cc_name 고객명 전달
0007 상품명누락 cc_pord_desc 상품명 전달
0008 결제금액누락 amount 숫자 형식 금액 전달
0009 전화번호누락 phone_no 숫자만 전달 (하이픈 제거)
0014 유효기간누락 expire_date YYMM 형식 전달
0015 할부개월수누락 install_month 00 또는 02~12 전달
0016 여행사명누락 agency_name SMS/KTK에서 sms_message 없을 시 필수
0018 예약번호누락 reservation_no SMS/KTK에서 sms_message 없을 시 필수

cancel 검증 에러

코드 메시지 원인 해결 방법
0010 가맹점터미널ID불일치 미등록 터미널ID KICC 발급 유효한 ID 사용
0011 거래번호중복 동일 주문번호 존재 새 주문번호로 재시도
0012 주문건수와 금액건수 불일치 orders 배열 필수 필드 누락 모든 항목에 필수 필드 포함
0013 주문건수와 상품명건수 불일치 orders 배열 필수 필드 누락 모든 항목에 필수 필드 포함
warning 배치 주문에서 일부라도 실패하면 전체 주문이 저장되지 않습니다. 모든 주문이 성공해야 정상 처리됩니다.
